#+title: Noctalia v5 settings snapshot

This directory is an *archive*, not a live module. Nothing here is imported or symlinked by =../default.nix= (which is pinned to Noctalia *v4*). It preserves the look-and-feel work done on the v5 alpha so it can be restored when v5 is ready for daily use.

* Why this exists
v4 and v5 store config completely differently, so the v5 tweaks can't just live in the v4 =settings.json=:

| Aspect        | v4 (active)                             | v5 (this snapshot)                      |
|---------------+-----------------------------------------+-----------------------------------------|
| Main settings | =~/.config/noctalia/settings.json= (JSON) | =~/.local/state/noctalia/settings.toml=   |
| Colors        | inline in settings.json                 | =~/.config/noctalia/colors.json=          |
| Plugins       | manual                                  | =~/.config/noctalia/plugins.json= + repos |
| Binary        | =noctalia-shell=                          | =noctalia=                                |

* Files
- =settings.toml= -- copied from =~/.local/state/noctalia/settings.toml=. Bar layout (Main bar: opacity, capsule, order, scale 0.8, thickness 24), dock (auto-hide), theme (wallpaper / m3-rainbow), per-widget tweaks (clock format, battery graphic, media, launcher glyph), lockscreen widget.
- =colors.json= -- copied from =~/.config/noctalia/colors.json=. Custom M3 palette (blue primary, dark surface).
- =plugins.json= -- copied from =~/.config/noctalia/plugins.json=. Plugin sources incl. =github:rytswd/noctalia-extra=, and per-plugin enabled state.

Captured on 2026-06-14 from the running v5 alpha instance.

* Host-specific values to review before restoring

- =settings.toml= references =eDP-1= (single internal display) and a lockscreen widget positioned for a 1728-wide output -- re-check per host.

* Restoring when v5 lands
1. Re-pin the input in =flake.nix=: =noctalia.url = "github:noctalia-dev/noctalia";= (drop the legacy-v4 ref), then =nix flake lock --update-input noctalia=.
2. Restore the v5 form of =../default.nix= (=programs.noctalia=, no out-of-store settings.json) -- it's preserved in git at commit =f3dc3bc "Adjust noctalia to be v5 ready"=.
3. Re-point the compositors at the v5 binary / IPC: niri =config.kdl= and hyprland =processes.conf= (=noctalia= + =noctalia msg ...=); also in =f3dc3bc=.
4. Seed the GUI state from this snapshot, then let the GUI own it:
   #+begin_src bash
     mkdir -p ~/.local/state/noctalia ~/.config/noctalia
     cp settings.toml ~/.local/state/noctalia/settings.toml
     cp colors.json   ~/.config/noctalia/colors.json
     cp plugins.json  ~/.config/noctalia/plugins.json
   #+end_src
   v5 writes runtime changes back to the state dir (Nix doesn't manage it), so this is a one-time seed -- refresh this snapshot from the live files if you want to re-capture later.
