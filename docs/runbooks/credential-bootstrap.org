#+title: Runbook: Credential bootstrap (full vs degraded eval)

This runbook decides one thing on a fresh machine: whether the first build of this flake is /full/ (the =nix-config-private= input is reachable over SSH) or /degraded/ (the public stub substitutes for it). The design and rationale live in [[file:../../air/v0.1/private-input-stub.org]]; this is just the operational path.

* Preflight

Run the shared preflight from a checkout of this repo; it probes the available credentials and prints the verdict with the exact commands.

#+begin_src bash
  bash apps/lib/preflight.sh
#+end_src

* Decision tree

** A helper machine is available
SSH into the new machine with agent forwarding (~ssh -A~) from a machine that already has GitHub access, so the forwarded agent satisfies the =git+ssh= input. See the access section of [[file:../../nixos-config/asus-rog-flow-z13-2025/README.org]] for the gpg-agent socket setup on the helper side. Re-run the preflight over that SSH session; it should report =full=, and no override is needed.

** Only a YubiKey is at hand
The FIDO2 resident keys carry the credential on the hardware itself; download them onto the new machine and load them into the agent.

#+begin_src bash
  cd ~/.ssh && ssh-keygen -K   # writes id_*_sk_rk* files; touch the key when prompted
  ssh-add ~/.ssh/id_ed25519_sk_rk*
#+end_src

Re-run the preflight; once it reports =full=, build normally.

** Neither -- degraded build
Legitimate state, not a dead end (e.g. waiting on hardware key delivery). Substitute the public stub for the private input; every host and home profile evaluates, with eval-time warnings announcing what is missing.

#+begin_src bash
  # From a checkout:
  nixos-rebuild build --flake .#<host> \
      --override-input nix-config-private path:./stubs/nix-config-private

  # Without any clone (GitHub tarball API, no SSH involved):
  nix build 'github:rytswd/nix-config#<attr>' \
      --override-input nix-config-private 'github:rytswd/nix-config?dir=stubs/nix-config-private'
#+end_src

What degraded means concretely: users get no private =hashedPassword= (check the host's =user-config/*/create.nix= fallback before relying on a login), no email accounts, no private git identity, and no sops secret material. The stub flake's comments ([[file:../../stubs/nix-config-private/flake.nix]]) are the authoritative per-attribute record.

* Upgrading degraded to full

Install a credential via either branch above, confirm with the preflight, then rebuild *without* the override -- nothing else changes. The first full build fetches =nix-config-private= at the locked revision and the eval warnings disappear; no cleanup of the degraded generation is needed beyond the rebuild itself.
